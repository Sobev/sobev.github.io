Mysql page查询优化

在千万级数据中，分页查询到第1000w之后的十条数据

首先想到的就是SELECT * FROM bill_in bi LIMIT 10000000,10

总共有1900w+数据
统计这些数据居然也花了11s，因为用的是Innodb，并没有维护总的数据条数，因此要一条条计数
这也从侧面说明了在大数据量的情况下，使用Count(*)是不太合适的

看看分页查询的执行时间 :)

Shit!! :(
时间也很长，返回前端的话也肯定已经超时了

优化手段

在这张表中，id是它的索引，并且设置成了自增，因此可以利用这个特性，使用索引加快速度

试试时间

3.7s，相比前面来说快了很多，由于服务器性能原因只能到这个水平了。。。
这3.7s的时间也大都花在了SELECT id FROM bill_in LIMIT 19000000,1这一句上
